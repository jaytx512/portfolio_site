# Working Agreements

## Commands
- Install: `npm install`
- Dev server: `npm run dev`
- Build: `npm run build`
- Preview build: `npm run preview`
- Lint: `npm run lint`
- Typecheck: `npm run typecheck`
- Format: `npm run format` or `npm run format:write`

## Code Style
- Use Prettier for formatting.
- Keep Tailwind class lists readable and grouped by layout -> spacing -> color.
- Prefer explicit names over abbreviations.

## Naming Conventions
- Components: `PascalCase`.
- Data files: `kebab-case` or `camelCase` (match existing).
- Utility functions: `camelCase`.

## Git
- Branches: `feature/<short-name>` or `fix/<short-name>`.
- Commits: short, imperative subject lines (e.g., "Add project cards").
